ชั่วโมงที่ 1: คิดแบบแยกปัญหา

Decomposition | แบ่งงานใหญ่ให้เป็นงานย่อยที่ทำได้จริง

เด็กต้องทำได้

  • บอกได้ว่างานใหญ่คืออะไร
  • แบ่งปัญหาใหญ่เป็นงานย่อย 4–6 ขั้นได้
  • จัดลำดับงานย่อยก่อน–หลังได้
  • ตรวจได้ว่างานย่อยมีส่วนใดซ้ำหรือตกหล่น
  • อธิบายเหตุผลว่าแบ่งแบบนี้เพราะอะไร

สื่อที่ใช้ได้

  • ภาพห้องเรียนรก / กระเป๋านักเรียน / งานวันเกิด / โต๊ะอ่านหนังสือ
  • ครูเปิดรูปในเน็ตหรือเล่าให้เด็กจินตนาการ
  • เด็กใช้สมุดทำตาราง งานใหญ่ → งานย่อย → ลำดับ
  • ไม่ต้องมีใบงานพิมพ์
ครูพูด/ย้ำ

วันนี้เราจะไม่แก้ปัญหาใหญ่ทีเดียว แต่จะแบ่งมันออกเป็นส่วนเล็ก ๆ เหมือนคอมพิวเตอร์ที่ทำงานทีละขั้น

แก่นชั่วโมงที่ 1: แบ่งปัญหาให้เล็กลง แล้วรวมกลับเป็นคำตอบของงานใหญ่

โครงเวลา ชั่วโมงที่ 1

60 นาที | เน้นจดในสมุดและอภิปรายจากภาพ

ลำดับกิจกรรม

  • 0–5 นาที: เปิดภาพงานยุ่ง ๆ แล้วถามว่างานใหญ่คืออะไร
  • 5–15 นาที: เกม “แยกงานใหญ่ให้เป็นงานย่อย”
  • 15–25 นาที: สอนแก่น Decomposition
  • 25–40 นาที: เกม “นักสืบปัญหา” จากภาพหรือสถานการณ์
  • 40–52 นาที: สมุดบันทึกแยกปัญหา
  • 52–60 นาที: ตัวอย่างเพิ่ม + Exit Ticket

สิ่งที่ครูต้องดูระหว่างสอน

  • เด็กแบ่งงานเป็นงานย่อยจริงหรือยัง
  • เด็กเขียนกว้างเกินไปไหม เช่น “ทำให้เสร็จ”
  • ลำดับงานก่อน–หลังสมเหตุสมผลไหม
  • เด็กอธิบายเหตุผลด้วยคำว่า “เพราะ” ได้ไหม
ครูพูด/ย้ำ

ถ้าเด็กตอบกว้างเกินไป ให้ถามว่า ‘แล้วต้องทำอะไรเป็นข้อแรก’ เพื่อบีบให้เล็กลง

ไม่ต้องรีบสอนศัพท์อังกฤษ ให้เด็กเข้าใจจากงานใกล้ตัวก่อน

05/27

เปิดชั่วโมง: งานใหญ่คืออะไร

ใช้ภาพในเน็ตหรือภาพในหัวเด็ก

ขั้นตอนครู

  • ครูเปิดภาพห้องรก โต๊ะเรียนรก หรือกระเป๋านักเรียนที่ของเยอะ
  • ถามว่า “ถ้าต้องจัดให้เรียบร้อย งานใหญ่คืออะไร”
  • รับคำตอบเด็กก่อน เช่น จัดห้อง จัดโต๊ะ จัดกระเป๋า
  • ถามต่อว่า “ถ้าจะเริ่มทำจริง ๆ ต้องทำอะไรก่อน”

เด็กทำอะไร

  • ดูภาพและบอกงานใหญ่
  • เสนอสิ่งที่ต้องทำอย่างน้อย 3 อย่าง
  • จดหัวข้อในสมุด: คิดแบบแยกปัญหา
  • แบ่งสมุดเป็น 3 ช่อง: งานใหญ่ / งานย่อย / ลำดับ
ครูพูด/ย้ำ

งานใหญ่บางอย่างดูยาก เพราะมันรวมหลายงานย่อยไว้ด้วยกัน ถ้าเราแยกออกมา เราจะเห็นว่าเริ่มตรงไหนได้

เด็กต้องเข้าใจ: งานใหญ่ต้องแตกเป็นงานย่อยก่อนลงมือ
🎮 ชั่วโมงที่ 1
06/27

เกมที่ 1: แยกงานใหญ่ให้เป็นงานย่อย

เกมจินตนาการจากภาพหรือสถานการณ์

วิธีเล่น

  • ครูให้สถานการณ์ เช่น “เตรียมกระเป๋าไปโรงเรียน”
  • เด็กเขียนงานใหญ่ไว้บนสุดของสมุด
  • เด็กแตกงานย่อยอย่างน้อย 5 ขั้น
  • วงงานที่ต้องทำก่อนด้วยสีหรือสัญลักษณ์
  • ครูสุ่มเด็กอ่าน แล้วเพื่อนช่วยถามว่ายังมีอะไรตกหล่นไหม

ตัวอย่างคำตอบ

  • งานใหญ่: เตรียมกระเป๋าไปโรงเรียน
  • งานย่อย: ดูตารางเรียน / เลือกหนังสือ / ใส่สมุด / ใส่ดินสอ / ตรวจการบ้าน / ปิดกระเป๋า
  • ลำดับที่ดี: ดูตารางเรียนก่อนเลือกหนังสือ
  • จุดที่ตกหล่นบ่อย: ลืมตรวจการบ้านหรืออุปกรณ์เขียน
ครูพูด/ย้ำ

ถ้าเขียนว่า ‘จัดกระเป๋า’ อย่างเดียว ยังไม่เล็กพอ ต้องบอกให้ได้ว่าข้างในมีขั้นตอนอะไรบ้าง

รูปแบบคำตอบที่ดี: ฉันแบ่งเป็น... เพราะ...

07/27

สอนแก่น: Decomposition คืออะไร

อธิบายให้เข้าใจด้วยภาษาง่าย

เนื้อหาที่ครูอธิบาย

  • Decomposition คือ การแยกปัญหาใหญ่ให้เป็นปัญหาย่อย
  • งานย่อยต้องเล็กพอที่จะลงมือทำได้
  • ต้องดูว่าแต่ละงานเกี่ยวข้องกันอย่างไร
  • เมื่อทำงานย่อยครบ จึงรวมกลับเป็นงานใหญ่ที่สำเร็จ
  • ปัญหาเดียวอาจแบ่งได้หลายแบบ แต่ต้องอธิบายเหตุผลได้

ให้เด็กจดในสมุด

  • งานใหญ่ = สิ่งที่ต้องการทำให้สำเร็จ
  • งานย่อย = ส่วนเล็ก ๆ ของงานใหญ่
  • ลำดับ = อะไรควรทำก่อน–หลัง
  • ตรวจ = มีอะไรซ้ำหรือตกหล่นไหม
ครูพูด/ย้ำ

คอมพิวเตอร์ทำงานเก่งเพราะมันทำตามขั้นเล็ก ๆ ที่ชัดเจน มนุษย์ก็คิดแบบนี้ได้เมื่อเจอปัญหายาก

คำจำง่าย: ใหญ่ → ย่อย → เรียง → ตรวจ → รวม

เกมที่ 2: นักสืบปัญหา

ใช้รูปในเน็ตหรือเรื่องเล่าให้เด็กคิด

วิธีเล่น

  • ครูเปิดภาพงานวุ่น ๆ เช่น งานวันเกิด ห้องเรียนก่อนจัดนิทรรศการ หรือโต๊ะที่มีของเยอะ
  • เด็กเขียนในสมุดว่า เป้าหมายคืออะไร
  • แตกงานย่อยอย่างน้อย 6 ข้อ
  • จัดหมวด เช่น คน / ของ / เวลา / สถานที่ / ตรวจความเรียบร้อย
  • เลือก 1 งานย่อยที่ควรทำก่อนที่สุด พร้อมเหตุผล

ตัวอย่าง: จัดงานวันเกิด

  • เป้าหมาย: จัดงานวันเกิดให้เสร็จ
  • งานย่อย: เชิญเพื่อน / เตรียมอาหาร / ตกแต่ง / เตรียมเกม / จัดโต๊ะ / เก็บขยะ
  • งานที่ควรทำก่อน: กำหนดจำนวนคน เพราะเกี่ยวกับอาหารและที่นั่ง
  • งานที่ทำท้าย: เก็บขยะและตรวจความเรียบร้อย
ครูพูด/ย้ำ

การแยกงานไม่ได้มีคำตอบเดียว แต่คำตอบต้องช่วยให้ทำงานง่ายขึ้นและตรวจได้

ถ้าเด็กทำได้ ให้ลองถามว่า “มีวิธีแบ่งแบบอื่นไหม”

Algorithm | คำสั่งต้องชัด ทำตามแล้วได้ผล

เด็กต้องทำได้

  • อธิบายได้ว่าอัลกอริทึมคือขั้นตอนแก้ปัญหา
  • เรียงลำดับขั้นตอนให้ถูกต้อง
  • แก้คำสั่งกำกวมให้ชัดเจน
  • ทดลองให้เพื่อนทำตามคำสั่งได้
  • ปรับคำสั่งจากผลการทดสอบ

สื่อที่ใช้ได้

  • ภาพกิจวัตร เช่น ล้างมือ แปรงฟัน พับกระดาษ เดินไปประตู
  • ครูเล่าให้เด็กจินตนาการว่าเป็นหุ่นยนต์
  • เด็กใช้สมุดเขียนลำดับขั้นตอน
  • ไม่ต้องมีเครื่องคอมพิวเตอร์
ครูพูด/ย้ำ

อัลกอริทึมที่ดีต้องชัดจนคนอื่นทำตามได้โดยไม่ต้องเดา ถ้ายังต้องเดา แปลว่าคำสั่งยังไม่ชัด

แก่นชั่วโมงที่ 2: คำสั่งชัด + ลำดับถูก + มีจุดเริ่มและจุดจบ

เกมที่ 1: หุ่นยนต์ทำตามคำสั่ง

ใช้ร่างกาย จินตนาการ และสมุดจด

วิธีเล่น

  • เลือกเด็ก 1 คนเป็นหุ่นยนต์ หรือครูเป็นหุ่นยนต์เอง
  • เด็กคนอื่นเขียนคำสั่ง 3–5 ขั้นในสมุด
  • หุ่นยนต์ทำตามคำสั่งแบบตรงตัว ห้ามเดาเพิ่ม
  • ถ้าทำไม่สำเร็จ ให้หาว่าคำสั่งใดกำกวม
  • เด็กแก้คำสั่งแล้วทดสอบใหม่

ตัวอย่างภารกิจ

  • เดินไปหยิบดินสอ
  • วางสมุดบนโต๊ะ
  • หันไปทางประตูแล้วเดิน 3 ก้าว
  • พับกระดาษครึ่งหนึ่ง
  • จัดดินสอ 3 แท่งให้เรียงจากสั้นไปยาว
ครูพูด/ย้ำ

เวลาเขียนคำสั่ง อย่าใช้คำว่า ‘ตรงนั้น’ หรือ ‘นิดหนึ่ง’ ให้บอกจำนวน ทิศทาง หรือจุดหมายให้ชัด

คำตอบที่ดี: เดินหน้า 2 ก้าว → หันขวา → หยิบดินสอ → กลับโต๊ะ

สอนแก่น: อัลกอริทึมคืออะไร

ขั้นตอนต้องมีลำดับและตรวจได้

เนื้อหาที่ครูอธิบาย

  • อัลกอริทึม คือ ลำดับขั้นตอนสำหรับแก้ปัญหา
  • ต้องมีจุดเริ่มต้นและจุดจบ
  • แต่ละขั้นต้องชัดเจนและทำได้
  • หากสลับลำดับ ผลลัพธ์อาจเปลี่ยน
  • อัลกอริทึมเดียวกันอาจมีหลายวิธี แต่ต้องถึงเป้าหมาย

เด็กจดในสมุด

  • เริ่มต้น: จะทำอะไร
  • ขั้นตอน: ทำอะไรก่อน–หลัง
  • ผลลัพธ์: เมื่อทำครบจะได้อะไร
  • ทดสอบ: คนอื่นทำตามได้หรือไม่
ครูพูด/ย้ำ

ถ้าเขียนว่า ‘แปรงฟัน’ คำเดียว ยังไม่ใช่อัลกอริทึมที่ละเอียดพอ ต้องแตกเป็นหยิบแปรง ใส่ยาสีฟัน แปรง บ้วนปาก และล้างแปรง

คำจำง่าย: เริ่ม → ทำทีละขั้น → ตรวจ → จบ

เกมที่ 2: แก้คำสั่งกำกวม

ครูให้คำสั่งไม่ชัด เด็กช่วยแก้ให้ชัด

วิธีเล่น

  • ครูเขียนคำสั่งกำกวมบนจอหรือกระดาน
  • เด็กจดในสมุดว่า “ไม่ชัดเพราะอะไร”
  • เด็กเขียนคำสั่งใหม่ให้ชัดกว่าเดิม
  • ให้เพื่อนลองทำตามคำสั่งใหม่
  • ถ้ายังผิด ให้ปรับอีกครั้ง

ตัวอย่างคำสั่งกำกวม

  • เดินไปนิดหนึ่ง → แก้เป็น เดินหน้า 2 ก้าว
  • วางไว้ตรงนั้น → แก้เป็น วางดินสอบนมุมขวาของโต๊ะ
  • จัดให้ดี → แก้เป็น เรียงสมุดจากเล่มใหญ่ไปเล่มเล็ก
  • หมุนหน่อย → แก้เป็น หมุนตัวไปทางขวา 90 องศา
ครูพูด/ย้ำ

คำสั่งที่ดีไม่ใช่คำสั่งที่ยาวที่สุด แต่เป็นคำสั่งที่ชัดพอให้ผู้อื่นทำตามได้ตรงกัน

ชั่วโมงที่ 3: เส้นทางบนตารางและพิกัด

Grid & Coordinate | วางแผนเส้นทางด้วยคำสั่ง

เด็กต้องทำได้

  • วาดตารางอย่างง่ายในสมุดได้
  • ระบุตำแหน่งด้วยแถวและคอลัมน์หรือ A1, B2 ได้
  • เขียนคำสั่งเดินหน้า หันซ้าย หันขวาได้
  • จำลองเส้นทางทีละคำสั่ง
  • ตรวจได้ว่าเส้นทางชนสิ่งกีดขวางหรือไม่

  • รูปแผนที่ตาราง เขาวงกต แผนผังห้องเรียน หรือเกมกระดาน
  • ครูวาดตารางบนกระดาน หรือเด็กวาดในสมุด
  • ใช้จินตนาการเป็นหุ่นยนต์เดินบนแผนที่
  • ไม่ต้องใช้คอมพิวเตอร์

เส้นทางที่ดีต้องไม่ใช่แค่ถึงเป้าหมาย แต่ต้องเดินตามคำสั่งได้จริงและไม่ชนสิ่งกีดขวาง

สอนแก่น: ตาราง พิกัด และทิศทาง

เนื้อหาที่ครูอธิบาย

  • ตารางช่วยแบ่งพื้นที่เป็นช่อง ๆ
  • พิกัดใช้บอกตำแหน่ง เช่น A1, B3
  • จุดเริ่มคือช่องที่หุ่นยนต์เริ่มอยู่
  • ทิศเริ่มคือหุ่นยนต์หันหน้าไปทางใด
  • หันซ้าย/หันขวาเปลี่ยนทิศ แต่ยังไม่เปลี่ยนตำแหน่ง

เด็กจดในสมุด

  • เดินหน้า = ย้ายไปช่องข้างหน้า 1 ช่อง
  • หันซ้าย = เปลี่ยนทิศไปทางซ้าย
  • หันขวา = เปลี่ยนทิศไปทางขวา
  • สิ่งกีดขวาง = ช่องที่ห้ามเดินผ่าน
ครูพูด/ย้ำ

หลายคนเข้าใจผิดว่า ‘หันขวา’ คือเดินไปทางขวา แต่จริง ๆ คือหมุนตัวเท่านั้น ยังไม่ขยับช่อง

คำจำง่าย: หัน = เปลี่ยนหน้า | เดิน = เปลี่ยนช่อง
